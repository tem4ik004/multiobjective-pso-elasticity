# Роевая многокритериальная оптимизация для задач теории упругости

Учебный пример реализации алгоритма ройной многокритериальной оптимизации (MOPSO) на C#.

## Постановка задачи

Цель работы — реализовать базовый вариант алгоритма Particle Swarm Optimization (PSO) в многокритериальной формулировке (MOPSO) и протестировать его на простой функции.

На данном этапе в качестве тестовой задачи используется минимизация функции

f(x, y) = x^2 + y^2,


x и y— два параметра. У функции один глобальный минимум в точке (0, 0)

В дальнейшем этот же алгоритм планируется применять к задачам теории упругости (оптимизация параметров конструкции по нескольким критериям: масса, напряжения, прогибы и т.д.).

## Реализация

Проект написан на C# (.NET).

Основные элементы:

- **Particle** — частица роя:
  - текущее положение в пространстве параметров;
  - скорость;
  - личное лучшее положение (personal best).
- **ArchiveMember** — элемент внешнего архива недоминируемых решений.
- **MOPSO** — класс, реализующий алгоритм:
  - инициализация роя случайными решениями;
  - вычисление целевой функции;
  - обновление личных best с учётом Парето-доминирования;
  - поддержание внешнего архива недоминируемых решений;
  - выбор лидеров на основе crowding distance;
  - обновление скоростей и положений частиц.

В текущей конфигурации используется одна целевая функция (минимизация x^2 + y^2), но интерфейс `objectiveFunction` допускает вектор целей (`double[]`), то есть алгоритм уже готов к многокритериальным задачам.

## Запуск

Требуется установленный .NET SDK.

В командной строке/терминале:

dotnet run
